# JSL Landing Page
## GitHub Collaboration Guideline (Team-Friendly)
### TL;DR
- Use branches like feature/* and fix/*.
- Make pull requests to dev.
- Use good commit messages (feat: add x).
- Open issues for all changes.
- Review PRs well.
- Keep code clean & documented.

### Commit messages
- "🐛 fix": Used to indicate that a specific bug or error has been fixed.

- "✨ add": Used to indicate that new code or functionality has been added to the project.

- "♻️ update": Used to indicate that existing code or functionality has been updated.

- "🗑️ remove": Used to indicate that code or functionality has been removed from the project.

- "refactor": Used to indicate that code has been restructured or rewritten in a way that does not change its functionality.

- "merge": Used to indicate that changes from one branch have been merged into another branch.

- "📚 docs": Used to indicate that documentation has been added, updated, or removed.

- "🎨 style": Used to indicate that code formatting or style conventions have been updated.

- "test": Used to indicate that tests have been added, updated, or removed.

- "📇 chore": Used to indicate that non-functional changes have been made, such as updating dependencies or build processes.

- "🚧 wip" : stands for "working in progress".

- "📦 build" : stands for "working in progress".
